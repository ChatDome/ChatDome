# 动态 Shell 命令使用规范

## 适用场景

当预定义命令（`run_security_check`）无法满足需求时，需要通过 `run_shell_command` 执行自定义命令。

## 工具优先级

必须遵循以下优先级：

1. **优先使用 `run_security_check`**：如果需求可以通过已有的 check_id 满足，禁止使用 `run_shell_command`
2. **只在确实无法匹配时使用 `run_shell_command`**

判别方法：检查 System Prompt 中的"可用的预定义命令 (check_id)"列表，看是否有匹配项。

## `run_shell_command` 参数规范

- `command`（必填，字符串）：要执行的命令
  - **必须是单行字符串**，不允许包含原始换行符或制表符
  - 多步操作使用 `;` 或 `&&` 连接
  - 示例：`cat /etc/os-release && uname -r`
  
- `reason`（必填，字符串）：执行理由
  - 此字段会展示给审批员看，必须清晰说明目的
  - 如果命令会修改系统，还必须说明影响范围和主要风险
  - 示例："查看操作系统版本和内核版本，用于诊断兼容性问题"

## 命令生成约束

### 普通模式（默认）
- 只允许生成只读查询命令
- 禁止生成写入、删除、修改、安装、重启类命令
- 违反此规则的命令会被安全策略拦截

### god_mode（unrestricted command 模式）
- 可以生成运维操作命令（安装、重启、配置修改等）
- 仍需在 reason 中说明目的、影响范围和主要风险
- 高风险命令仍需用户审批确认

## 常用命令模式

### 系统信息类（安全）
- `cat /etc/os-release` — 操作系统版本
- `uname -r` — 内核版本
- `uptime` — 运行时间
- `df -h` — 磁盘使用
- `free -h` — 内存使用

### 网络诊断类（安全）
- `ss -tlnp` — 监听端口
- `ip addr show` — 网络接口
- `curl -s ifconfig.me` — 公网 IP

### 日志查询类（安全）
- `journalctl -u sshd -n 20 --no-pager` — SSH 服务日志
- `tail -100 /var/log/auth.log` — 认证日志

## 禁止事项

- 不要在命令中嵌入原始换行符，必须使用 `;` 或 `&&` 连接
- 不要生成交互式命令（如 `vim`、`nano`、`top` 不带 `-b`）
- 不要在普通模式下尝试写入或修改操作
- 不要为了绕过审批而拆分高危命令为多个低危命令

## 回答要求

展示命令执行结果时，先说明执行了什么命令及其目的，再分析输出内容。如果命令失败（非零退出码），说明可能的原因。
