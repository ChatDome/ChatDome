# 用户意图消歧指南

## 适用场景

用户提出的请求主体不明确、范围模糊，或可能匹配多个不同的操作手册章节时，需要先消歧再执行。

如果用户当前输入不足以确认是否要继续历史任务，且下一步会执行工具或命令，必须先追问确认。不要把低信息量回复自动解释为继续上一个未完成任务。

## 消歧决策树

### "最近执行了哪些命令"

用户没有指明主体时，必须追问：

```
"最近执行了哪些命令"
├── 主体是 ChatDome → command_audit 章节
├── 主体是 SSH 用户 → ssh_session_commands 章节
├── 主体是整台主机 → host_exec_audit 章节
└── 不确定 → 追问："您想查看的是 ChatDome 助手自己执行的命令、某个 SSH 登录用户的命令，还是整台服务器的全部命令历史？"
```

### "检查一下安全状况"

这是一个常见的宽泛请求，按以下组合执行快速安全总览：

1. `run_security_check` → `ssh_bruteforce`（SSH 暴力破解）
2. `run_security_check` → `open_ports`（异常端口监听）
3. `run_security_check` → `active_connections`（可疑连接）
4. `run_security_check` → `disk_usage`（磁盘异常）

执行后汇总各项结果，按风险程度排序报告。

### "这个 IP 是什么"

```
"这个 IP 是什么"
├── 用户提供了具体 IP → whois_lookup 查询归属
├── 来自 Sentinel 告警上下文 → 先查归属，再结合告警分析
└── 没有具体 IP → 追问："请提供要查询的 IP 地址"
```

### "为什么一直提醒"

```
"为什么一直提醒"
├── 确认是 Sentinel 告警 → sentinel 章节（查看状态机升级情况）
├── 确认是审批提醒 → approvals 章节
└── 不确定 → 追问具体是哪类提醒
```

## 追问模板

当需要追问时，使用简洁直接的格式：

> "您想查看的是以下哪种？\n1. ChatDome 助手自己执行的命令\n2. 某个 SSH 登录用户在会话中执行的命令\n3. 整台服务器的全部命令历史"

## 禁止事项

- 不要在主体不明确时猜测用户意图并直接执行
- 不要对模糊请求调用多个不相关的工具进行盲目试探
- 不要在追问后仍不确定时强行选择一个工具执行
- 不要在用户未明确同意时续跑历史任务中的工具调用

## 回答要求

追问时使用中文，简洁明了，提供 2-3 个明确选项让用户选择。
