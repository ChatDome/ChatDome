# IP 威胁情报查询

## 适用场景

用户要求查询某个 IP 的归属信息、地理位置，或 Sentinel 告警中涉及的来源 IP 需要情报补充。

## 优先工具

使用 `whois_lookup`。

### 参数说明

- `ip`（必填，字符串）：要查询的 IP 地址

### 返回值格式

```
IP: 203.0.113.42
国家: 中国
地区: 广东省
城市: 深圳市
ISP: China Telecom
组织: Chinanet
AS: AS4134 CHINANET-BACKBONE
```

### 错误情况

- `IP 查询失败: ...`：IP 格式无效或 API 返回错误
- `IP 查询超时: ...`：ip-api.com 网络不可达（10 秒超时）
- `IP 查询异常: ...`：其他运行时错误

## 与 Sentinel 告警联动

当 Sentinel 推送 SSH 暴力破解或可疑登录告警时，如果告警中包含来源 IP，应主动调用 `whois_lookup` 补充情报：

1. 从告警文本中提取来源 IP
2. 调用 `whois_lookup` 查询归属
3. 将情报整合到告警分析中

示例回答结构：
> 攻击来源 IP 203.0.113.42 归属于中国广东省深圳市，ISP 为 China Telecom（AS4134）。该 IP 在过去 10 分钟内发起了 47 次 SSH 登录失败尝试。

## 禁止事项

- 不要仅根据国家或地区判断 IP 是否恶意（同一国家的 IP 不一定都是攻击者）
- 不要对内网 IP（10.x.x.x、172.16-31.x.x、192.168.x.x）调用此工具
- API 数据仅供参考，不能替代专业威胁情报平台的判断

## 回答要求

展示查询结果时使用结构化格式，并结合告警上下文给出风险评估。
